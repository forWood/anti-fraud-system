## 1. 系统概述

### 1.1 系统简介

反欺诈与反恐融资风险识别智能体系统是银行风险管理体系的核心工具，实现从交易监控、风险评估到预警处置、案例调查、可疑报告上报的全流程闭环管理。

### 1.2 核心功能

| 功能模块 | 说明 |
|----------|------|
| **实时交易监控** | 全渠道交易实时评分，识别可疑交易 |
| **智能预警** | 自动生成预警，推送给风控专员 |
| **案例调查** | 调查工作流，关联交易/客户/历史案例 |
| **AI辅助决策** | 知识库检索、相似案例推荐、处置建议 |
| **可疑报告** | 自动生成符合监管格式的STR报告 |
| **名单管理** | 黑名单/制裁名单维护与命中追溯 |
| **统计分析** | 风险监控大屏、模型效果监控、报表 |

### 1.3 风险等级说明

| 风险等级 | 评分范围 | 标识色 | 处置要求 | 响应时限 |
|----------|----------|--------|----------|----------|
| **高风险** | 80-100分 | 🔴 红色 | 立即预警，人工审核，必要时暂停交易 | ≤24小时 |
| **中风险** | 50-79分 | 🟡 黄色 | 预警，跟踪关注，需人工复核 | ≤72小时 |
| **低风险** | 0-49分 | 🟢 绿色 | 记录存档，持续监控 | 定期复核 |

### 1.4 系统决策说明

| 决策 | 含义 | 后续操作 |
|------|------|----------|
| **PASS (放行)** | 交易风险可控 | 正常完成，记录监控 |
| **REVIEW (审核)** | 交易存在可疑特征 | 生成预警，人工复核 |
| **BLOCK (阻断)** | 交易风险较高 | 立即阻断，强制人工审核 |

## 5. 预警处理流程

### 5.1 预警工作台

**入口：** 左侧菜单 → 预警工作台

**预警列表字段：**
| 字段 | 说明 |
|------|------|
| 预警编号 | ALT开头的唯一ID |
| 风险等级 | 🔴高 / 🟡中 / 🟢低 |
| 风险评分 | 0-100分 |
| 交易金额 | 关联交易金额 |
| 创建时间 | 预警生成时间 |
| 处理截止 | 超时升级提醒 |
| 状态 | 待处理/处理中/已解决/已关闭 |
| 处理人 | 当前处理人 |

### 5.2 处理预警

**Step 1: 认领预警**
1. 在待处理列表中找到预警
2. 点击"认领"按钮
3. 预警状态变为"处理中"

**Step 2: 查看详情**
1. 点击预警编号查看详情
2. 查看关联交易信息
3. 查看AI智能体决策解释：
   - 命中规则及原因
   - 相似历史案例
   - ML模型主要判定依据
4. 查看客户历史行为数据

**Step 3: 做出处置**
```
处置选项:
☐ 确认为风险 → 升级为案例，启动调查
☐ 排除风险   → 标记为误报，提交复核
☐ 信息不足   → 请求补充信息
☐ 转发       → 转发给其他同事
```

**Step 4: 填写处理意见**
```
处理意见模板:
1. 风险判定: [确认为可疑 / 排除风险]
2. 判定依据: [命中规则/客户行为/人工判断]
3. 处置操作: [账户冻结/电话确认/持续监控]
4. 备注: [其他说明]
```

### 5.3 预警升级机制

| 场景 | 升级条件 | 操作 |
|------|----------|------|
| 超时未处理 | 高风险24h / 中风险72h | 自动升级到上一级 |
| 风险评分极高 | 评分≥95 | 直接通知风控经理 |
| 制裁名单命中 | T001规则触发 | 立即通知合规部门 |
| 集群预警 | 同商户/同IP多预警 | 合并升级为团伙风险 |

**通知渠道：**

## 6. 案例调查流程

### 6.1 创建案例

**方式一：从预警创建**
1. 在预警详情页点击"升级为案例"
2. 系统自动填充交易信息和预警数据
3. 选择案例类型和调查模板
4. 点击"创建"生成案例

**方式二：直接创建**
1. 左侧菜单 → 案例管理 → 新建案例
2. 填写案例信息：
   - 案例标题
   - 案例类型 (欺诈/洗钱/恐融)
   - 关联客户/账户
   - 关联交易列表
3. 保存草稿

### 6.2 调查流程

```
┌────────┐   ┌────────┐   ┌────────┐   ┌────────┐   ┌────────┐
│ 信息收集 │→  │ 分析评估 │→  │ 形成结论 │→  │ 撰写报告 │→  │ 审核归档 │
└────────┘   └────────┘   └────────┘   └────────┘   └────────┘
   草稿       调查中       初步结论     报告编写       已审核
```

**信息收集阶段：**
- 调取关联交易明细
- 查看客户基本信息
- 查询历史交易行为
- 知识库检索相似案例
- 导入外部信息 (工商/司法)

**分析评估阶段：**
- 分析交易链路图 (可视化)
- 识别账户关联关系 (图谱)
- 评估资金流向 (来源/去向)
- AI辅助分析 (智能体建议)

**形成结论：**
- 判定可疑类型 (STR01-STR99)
- 评估风险等级
- 拟定处置建议
- 填写调查摘要

### 6.3 调查报告

**报告模板结构：**
```
案例调查报告
━━━━━━━━━━━━━━━━
1. 基本信息
   - 案例编号 / 类型 / 创建时间
   - 调查人 / 审核人

2. 客户信息
   - 客户编号 / 姓名 / 证件类型
   - 开户日期 / 风险等级
   - 职业 / 行业

3. 交易分析
   - 可疑交易明细 (表格)
   - 交易链路分析
   - 资金流向图

4. 风险判定
   - 命中规则列表
   - 相似案例引用
   - AI风险评估

5. 调查结论
   - 可疑类型判定
   - 风险等级评估
   - 处置建议

6. 附件
   - 交易明细
   - 证明材料
```

## 7. 可疑交易报告 (STR)

### 7.1 自动生成

**触发条件：**
- 案例调查结论为"确认为可疑交易"
- 风控经理审核通过

**生成流程：**
1. 系统自动提取案例信息
2. 填充人民银行标准STR模板
3. 生成XML格式报告
4. 审核人预览确认

### 7.2 报告内容

**必填字段：**
| 字段 | 来源 |
|------|------|
| 报告机构代码 | 系统配置 |
| 客户姓名/证件号 | 客户信息系统 |
| 交易时间/金额 | 交易记录 |
| 可疑类型 | 案例调查结论 |
| 可疑特征描述 | 案例调查报告 |
| 处置措施 | 处置建议 |

### 7.3 上报流程

```
案例审核通过 → 生成STR → 合规审核 → 上报监管 → 状态追踪
                  │                                 │
                  └── 退回修改 ←── 审核不通过 ←─────┘
```

**上报状态：**
- 草稿: 未完成
- 待审核: 等待合规审核
- 已上报: 已发送监管
- 已受理: 监管确认接收
- 已退回: 需要修改

## 8. 名单管理

### 8.1 名单类型

| 列表类型 | 来源 | 更新频率 | 说明 |
|----------|------|----------|------|
| 制裁名单 | 联合国/中国 | 每日 | 命中立即阻断 |
| 内部黑名单 | 历史欺诈案例 | 实时 | 内部标记 |
| 高风险名单 | 监管下发 | 每周 | 加强尽职调查 |
| PEP名单 | 政治公众人物 | 每月 | 加强监控 |

### 8.2 名单匹配

**匹配方式：**
- 精准匹配: 账号/证件号SHA-256哈希
- 模糊匹配: 姓名拼音相似度 ≥ 80%

**命中处理：**
```
精准命中 → 立即预警 + 自动阻断
模糊匹配 → 生成待确认预警 → 人工核实
```

### 8.3 名单维护

**添加条目：**
1. 进入名单管理 → 对应名单类型
2. 点击"新增"
3. 填写:
   - 实体类型 (账号/姓名/证件/手机/IP)
   - 实体值 (系统自动脱敏)
   - 来源说明
   - 生效时间
4. 提交审批

**批量导入：**
1. 下载导入模板 (CSV)
2. 按模板填写数据
3. 上传文件
4. 预览确认
5. 执行导入

### Q2: 预警和案例有什么区别？

| 维度 | 预警 | 案例 |
|------|------|------|
| 产生方式 | 系统自动生成 | 人工从预警升级或直接创建 |
| 阶段 | 初步筛查 | 深度调查 |
| 操作 | 确认/排除/转发 | 信息收集/分析/报告 |
| 输出 | 处置结论 | 调查报告 |

---

### Q3: AI决策结果可信吗？如何参考？

AI智能体提供**辅助决策**，非最终决策。参考建议：
1. **PASS决策：** 通常可直接放行，但建议抽查
2. **REVIEW决策：** 必须人工复核，AI提供参考依据
3. **BLOCK决策：** 建议采纳，需人工最终确认

**查看AI决策依据：** 详情页 → 查看命中规则、相似案例、ML特征贡献

### Q4: 如何反馈规则误报？

**操作流程：**
1. 处理预警时选择"排除风险"
2. 在备注中说明误报原因
3. 提交后，系统自动记录误报数据
4. 风控经理定期审查误报率
5. 对高频误报规则进行调整

### Q8: 系统出现异常怎么处理？

| 异常 | 表现 | 处理 |
|------|------|------|
| 交易评分超时 | 交易长时间无响应 | 联系运维(分机:6001) |
| 知识库检索失败 | 无相似案例返回 | 联系模型团队(分机:6002) |
| 预警未生成 | 高风险交易无预警 | 检查Kafka/Agent状态 |
| 页面加载慢 | 列表加载超3秒 | 清理浏览器缓存 |

**紧急运维热线：** 6001 (7×24小时)
