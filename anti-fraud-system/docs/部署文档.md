## 1. 部署架构概述

### 1.1 部署模式

| 模式 | 适用场景 | 节点数 | 高可用 |
|------|----------|--------|--------|
| 单机Docker Compose | 开发/功能测试 | 1 | 无 |
| 多机Docker Compose | 集成测试/UAT | 2-3 | 部分 |
| Kubernetes集群 | 生产环境 | ≥3 | 完整 |

### 1.2 服务部署关系

```
┌──────────────────── Kubernetes Cluster (3 Node) ──────────────────────┐
│                                                                       │
│  Node-1:                          Node-2:        Node-3:             │
│  ┌──────────────────┐            ┌────────────┐  ┌────────────┐     │
│  │AI Agent (主)      │            │AI Agent(备) │  │AI Agent(备) │     │
│  │Knowledge Base(主) │            │Knowledge(备)│  │Knowledge(备)│     │
│  │TM/AM x2           │            │TM/AM x2     │  │CM/RG x2    │     │
│  │Nacos x2           │            │Nacos x1     │  │Prometheus  │     │
│  └──────────────────┘            └────────────┘  └────────────┘     │
│                                                                       │
│  ┌─ 中间件集群 ──────────────────────────────────────────────────┐   │
│  │ [Kafka x3]  [MySQL Master-Slave]  [Redis Sentinel] [ClickHouse] │   │
│  │ [ZooKeeper x3]  [Flink JM+TM]  [Milvus]  [Grafana]            │   │
│  └────────────────────────────────────────────────────────────────┘   │
└───────────────────────────────────────────────────────────────────────┘
```

## 2. 硬件与网络要求

### 2.1 生产环境硬件规格

| 节点类型 | 数量 | CPU | 内存 | 系统盘 | 数据盘 | 网络 |
|----------|------|-----|------|--------|--------|------|
| Master节点 | 3 | 4核 | 16GB | 100GB SSD | - | 10Gbps |
| 计算节点 | 6 | 16核 | 64GB | 200GB SSD | 1TB SSD | 10Gbps |
| 存储节点 | 3 | 8核 | 32GB | 200GB SSD | 2TB SSD | 10Gbps |

### 2.2 网络要求

| 网络段 | 用途 | 要求 |
|--------|------|------|
| 内网管理段 | K8s管理/Kafka/MySQL | 10.0.0.0/16, 低延迟 |
| 业务服务段 | 微服务间通信 | 10.1.0.0/16, 低延迟 |
| DMZ段 | API Gateway对外 | 172.16.0.0/16, 安全隔离 |
| 存储网络 | Ceph/分布式存储 | 独立VLAN, 10Gbps |

## 3. 环境准备

### 3.1 操作系统

```bash
# 推荐操作系统
CentOS 7.9+ / Rocky Linux 8+ / Ubuntu 20.04+

# 基础依赖
yum install -y wget curl git vim net-tools
# 或
apt-get install -y wget curl git vim net-tools
```

### 3.2 Docker安装

```bash
# CentOS/Rocky
yum install -y yum-utils
yum-config-manager --add-repo https://download.docker.com/linux/centos/docker-ce.repo
yum install -y docker-ce docker-ce-cli containerd.io docker-compose-plugin

# Ubuntu
apt-get update
apt-get install -y docker.io docker-compose-v2

# 启动Docker
systemctl enable docker --now

# 验证
docker --version          # ≥ 24.0.0
docker compose version    # ≥ 2.20.0
```

### 3.3 Kubernetes安装 (kubeadm)

```bash
# 1. 安装kubeadm/kubelet/kubectl
cat <<EOF > /etc/yum.repos.d/kubernetes.repo
[kubernetes]
name=Kubernetes
baseurl=https://pkgs.k8s.io/core:/stable:/v1.27/rpm/
enabled=1
gpgcheck=1
EOF
yum install -y kubeadm-1.27.* kubelet-1.27.* kubectl-1.27.*

# 2. 初始化Master节点
kubeadm init --pod-network-cidr=10.244.0.0/16 \
  --service-cidr=10.96.0.0/12 \
  --control-plane-endpoint=k8s-api.internal.com

# 3. 安装网络插件(Calico)
kubectl apply -f https://raw.githubusercontent.com/projectcalico/calico/v3.26/manifests/calico.yaml

# 4. 加入Worker节点
kubeadm join k8s-api.internal.com:6443 --token <token> --discovery-token-ca-cert-hash <hash>

# 5. 验证集群
kubectl get nodes
# NAME     STATUS   ROLES           AGE   VERSION
# master1  Ready    control-plane   5m    v1.27.10
# worker1  Ready    <none>          2m    v1.27.10
# worker2  Ready    <none>          2m    v1.27.10
```

### 3.4 Helm安装

```bash
curl https://raw.githubusercontent.com/helm/helm/main/scripts/get-helm-3 | bash
helm version  # ≥ 3.13
```

### 3.5 镜像仓库

```bash
# 使用Harbor私有镜像仓库
docker login harbor.bank.com -u robot$risk -p <token>

# 构建并推送镜像
cd anti-fraud-system
docker build -t harbor.bank.com/risk/anti-fraud-agent:2.0.0 \
  -f anti-fraud-deployment/docker/agent/Dockerfile .
docker push harbor.bank.com/risk/anti-fraud-agent:2.0.0

docker build -t harbor.bank.com/risk/knowledge-base:2.0.0 \
  -f anti-fraud-deployment/docker/knowledge/Dockerfile .
docker push harbor.bank.com/risk/knowledge-base:2.0.0
```

### 3.6 TLS证书

```bash
# 生成自签名证书 (开发环境)
openssl req -x509 -newkey rsa:4096 -keyout key.pem -out cert.pem \
  -days 365 -nodes -subj "/CN=anti-fraud.bank.com"

# 生产环境使用银行内部CA签发
# 将证书部署到K8s Secret
kubectl create secret tls anti-fraud-tls \
  --cert=cert.pem --key=key.pem -n risk-management
```

## 4. Docker Compose部署

### 4.1 快速部署（单机）

```bash
# 1. 克隆项目
git clone https://github.com/forWood/ozon.git anti-fraud-system
cd anti-fraud-system

# 2. 构建JAR包
mvn clean package -DskipTests

# 3. 启动所有服务
cd anti-fraud-deployment
docker compose up -d

# 4. 查看服务状态
docker compose ps

# 预期输出:
# NAME              STATUS          PORTS
# risk-zk           Up (healthy)    0.0.0.0:2181->2181/tcp
# risk-kafka        Up (healthy)    0.0.0.0:9092->9092/tcp
# risk-mysql        Up (healthy)    0.0.0.0:3306->3306/tcp
# risk-redis        Up (healthy)    0.0.0.0:6379->6379/tcp
# risk-clickhouse   Up              0.0.0.0:8123->8123/tcp
# risk-prometheus   Up              0.0.0.0:9090->9090/tcp
# risk-grafana      Up              0.0.0.0:3000->3000/tcp
# risk-agent        Up              0.0.0.0:8080->8080/tcp
# risk-knowledge    Up              0.0.0.0:8088->8088/tcp
# risk-tm           Up              0.0.0.0:8081->8081/tcp

# 5. 验证
curl http://localhost:8080/actuator/health
# {"status":"UP","components":{"db":{"status":"UP"},"redis":{"status":"UP"}}}
```

### 4.2 分步部署（多机）

```bash
# === 机器1: 基础设施 ===
docker compose -f docker-compose-infra.yml up -d
# 包含: ZK, Kafka, MySQL, Redis, ClickHouse

# === 机器2: 计算层 ===
docker compose -f docker-compose-compute.yml up -d
# 包含: Flink JM + TM

# === 机器3: 应用层 ===
docker compose -f docker-compose-app.yml up -d
# 包含: Agent, KB, 微服务
```

### 4.3 环境变量配置

```bash
# .env 文件
# === 数据库 ===
MYSQL_HOST=mysql
MYSQL_PORT=3306
MYSQL_USER=risk_user
MYSQL_PASSWORD=risk_pass_2026

# === Redis ===
REDIS_HOST=redis
REDIS_PORT=6379

# === Kafka ===
KAFKA_BOOTSTRAP_SERVERS=kafka:9092

# === ML服务 ===
ML_SERVICE_URL=http://tf-serving:8501

# === 知识库 ===
KB_SERVICE_URL=http://knowledge-base:8088

# === Agent配置 ===
RULE_WEIGHT=0.4
ML_WEIGHT=0.6
HIGH_RISK_THRESHOLD=80
```

### 4.4 生产环境独立配置

```yaml
# docker-compose.prod.yml (生产覆盖配置)
services:
  anti-fraud-agent:
    deploy:
      replicas: 3
      resources:
        limits:
          cpus: '2'
          memory: 4G
        reservations:
          cpus: '1'
          memory: 2G
    restart: always
    logging:
      driver: "json-file"
      options:
        max-size: "100m"
        max-file: "10"

  mysql:
    deploy:
      placement:
        constraints: [node.role == manager]
    volumes:
      - mysql-prod-data:/var/lib/mysql

  redis:
    deploy:
      replicas: 1
    command: redis-server --requirepass ${REDIS_PASSWORD} --appendonly yes
```

## 5. Kubernetes部署

### 5.1 命名空间创建

```bash
kubectl create namespace risk-management
kubectl label namespace risk-management \
  pod-security.kubernetes.io/enforce=baseline
```

### 5.2 Secrets配置

```bash
# 数据库密码
kubectl create secret generic anti-fraud-db-secret \
  --from-literal=host=mysql.risk-management.svc.cluster.local \
  --from-literal=username=risk_user \
  --from-literal=password='risk_prod_pass_2026' \
  -n risk-management

# Redis密码
kubectl create secret generic anti-fraud-redis-secret \
  --from-literal=host=redis.risk-management.svc.cluster.local \
  --from-literal=password='redis_prod_pass_2026' \
  -n risk-management

# API Key
kubectl create secret generic anti-fraud-api-key \
  --from-literal=api-key='sk-prod-xxxxxxxxxxxx' \
  -n risk-management

# 镜像拉取
kubectl create secret docker-registry harbor-secret \
  --docker-server=harbor.bank.com \
  --docker-username=robot$risk \
  --docker-password=<robot-token> \
  -n risk-management
```

### 5.3 Helm部署

```bash
# 添加Helm仓库 (如果有)
helm repo add bank-risk https://harbor.bank.com/chartrepo/risk
helm repo update

# 安装 (使用本地Chart)
cd anti-fraud-deployment/kubernetes
helm install anti-fraud ./helm-chart \
  --namespace risk-management \
  --create-namespace \
  --set global.environment=production \
  --set antiFraudAgent.replicaCount=3 \
  --set antiFraudAgent.resources.limits.cpu=2 \
  --set antiFraudAgent.resources.limits.memory=4Gi

# 查看部署状态
helm list -n risk-management
kubectl get pods -n risk-management

# 查看部署详情
kubectl describe deployment anti-fraud-agent -n risk-management
```

### 5.4 升级部署

```bash
# 镜像更新
helm upgrade anti-fraud ./helm-chart \
  --namespace risk-management \
  --set antiFraudAgent.image.tag=2.0.1

# 或直接更新Deployment
kubectl set image deployment/anti-fraud-agent \
  agent=harbor.bank.com/risk/anti-fraud-agent:2.0.1 \
  -n risk-management

# 查看滚动更新状态
kubectl rollout status deployment/anti-fraud-agent -n risk-management

# 如果更新出问题，回滚
kubectl rollout undo deployment/anti-fraud-agent -n risk-management
```

## 6. 配置说明

### 6.1 环境变量完整列表

| 变量名 | 默认值 | 必需 | 说明 |
|--------|--------|------|------|
| `SPRING_PROFILES_ACTIVE` | dev | 是 | 环境: dev/test/prod |
| `MYSQL_HOST` | localhost | 是 | MySQL地址 |
| `MYSQL_PORT` | 3306 | 是 | MySQL端口 |
| `MYSQL_USER` | risk_user | 是 | MySQL用户 |
| `MYSQL_PASSWORD` | - | 是 | MySQL密码 |
| `REDIS_HOST` | localhost | 是 | Redis地址 |
| `REDIS_PORT` | 6379 | 否 | Redis端口 |
| `REDIS_PASSWORD` | - | 否 | Redis密码 |
| `KAFKA_BOOTSTRAP_SERVERS` | localhost:9092 | 是 | Kafka地址 |
| `ML_SERVICE_URL` | - | 是 | TF Serving地址 |
| `KB_SERVICE_URL` | - | 否 | 知识库服务地址 |
| `DROOLS_RULES_PATH` | classpath:rules | 否 | 规则文件路径 |
| `RULE_WEIGHT` | 0.4 | 否 | 规则权重 |
| `ML_WEIGHT` | 0.6 | 否 | 模型权重 |
| `HIGH_RISK_THRESHOLD` | 80 | 否 | 高风险阈值 |
| `MEDIUM_RISK_THRESHOLD` | 50 | 否 | 中风险阈值 |

### 6.2 JVM参数推荐

| 环境 | JVM参数 |
|------|---------|
| 开发 | `-Xms512m -Xmx1g -XX:+UseG1GC` |
| 测试 | `-Xms1g -Xmx2g -XX:+UseG1GC -XX:MaxGCPauseMillis=200` |
| 生产 | `-Xms2g -Xmx4g -XX:+UseG1GC -XX:MaxGCPauseMillis=100 -XX:+HeapDumpOnOutOfMemoryError` |

### 6.3 配置文件优先级

```
高优先级: 环境变量 (K8s Secret/ConfigMap)
    ↓
中优先级: application-{profile}.yml
    ↓
低优先级: application.yml (默认值)
```

## 7. 安全管理

### 7.1 密码管理

```bash
# 禁止明文密码，使用K8s Secret
kubectl create secret generic db-credentials \
  --from-literal=password='<StrongP@ssw0rd>' \
  -n risk-management

# 在Pod中引用
env:
- name: MYSQL_PASSWORD
  valueFrom:
    secretKeyRef:
      name: db-credentials
      key: password
```

### 7.2 网络策略

```yaml
# 仅允许Agent访问MySQL
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: mysql-access
  namespace: risk-management
spec:
  podSelector:
    matchLabels:
      app: mysql
  ingress:
  - from:
    - podSelector:
        matchLabels:
          app: anti-fraud-agent
    ports:
    - port: 3306
```

### 7.3 Pod安全上下文

```yaml
securityContext:
  runAsNonRoot: true
  runAsUser: 1000
  runAsGroup: 1000
  fsGroup: 1000
  capabilities:
    drop: ["ALL"]
  readOnlyRootFilesystem: true
```

## 8. 扩容与缩容

### 8.1 自动扩容 (HPA)

```yaml
# 已内置在Helm Chart中
autoscaling:
  enabled: true
  minReplicas: 3
  maxReplicas: 10
  targetCPU: 70%
  targetMemory: 80%
```

### 8.2 手动扩容

```bash
# Agent扩容到5副本
kubectl scale deployment anti-fraud-agent --replicas=5 -n risk-management

# Transaction Monitor扩容到3副本
kubectl scale deployment transaction-monitor --replicas=3 -n risk-management

# Flink TaskManager扩容
kubectl scale deployment flink-taskmanager --replicas=4 -n risk-management
```

### 8.3 缩容顺序

```
1. Transaction Monitor → 3副本 (逐步)
2. Alert Management → 2副本
3. AI Agent → 3副本 (先缩备节点)
4. Knowledge Base → 2副本
```

## 9. 回滚策略

### 9.1 应用回滚

```bash
# 查看历史版本
helm history anti-fraud -n risk-management

# 回滚到上一版本
helm rollback anti-fraud -n risk-management

# 回滚到指定版本
helm rollback anti-fraud 2 -n risk-management

# Kubectl回滚
kubectl rollout undo deployment/anti-fraud-agent -n risk-management
kubectl rollout undo deployment/anti-fraud-agent --to-revision=3 -n risk-management
```

### 9.2 数据库回滚

```bash
# 应用flyway/liquibase进行数据库版本管理
# 回滚到上一版本
mvn flyway:undo -pl anti-fraud-sql

# 或从备份恢复
mysql -h localhost -u risk_user -p risk_db < /backup/anti-fraud/20260514/risk_db.sql
```

### B. Docker Compose vs K8s 选择

| 场景 | 推荐方案 | 理由 |
|------|----------|------|
| 个人开发 | Docker Compose | 轻量、快速启动 |
